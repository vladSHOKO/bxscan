# bxscan

CLI-инструмент для статического анализа проектов на **1С-Битрикс**, написанный на Go.

---

# Описание проекта

`bxscan` — консольная утилита, предназначенная для анализа структуры и качества Bitrix-проектов.

Основная задача инструмента — помочь разработчику быстро разобраться в незнакомом проекте, выявить потенциальные проблемы и проверить соблюдение базовых практик разработки.

В отличие от универсальных анализаторов кода (PHPStan, SonarQube и т.д.), `bxscan` ориентирован исключительно на особенности экосистемы **1С-Битрикс**.

---

# Цели проекта

* Практическое изучение языка Go.
* Создание реально полезного инструмента для ежедневной работы.
* Автоматизация первичного аудита Bitrix-проектов.
* Получение опыта разработки CLI-приложений.

---

# Целевая аудитория

* PHP-разработчики на Bitrix;
* новые разработчики, которым необходимо быстро изучить структуру проекта;
* тимлиды, проводящие технический аудит;
* разработчики, принимающие проекты на поддержку.

# Область анализа

На текущем этапе анализируется пользовательская часть проекта — директория:

```
./local
```

Инструмент работает со следующими разделами:

```
/local/components
/local/modules
/local/ajax
/local/php_interface
/local/templates
```

По умолчанию игнорируются директории:

```
vendor/
cache/
logs/
upload/
```

# MVP

Первая версия должна отвечать на три основных вопроса.

## 1. Из чего состоит проект?

Пример вывода:

```
Components: 14
Modules: 2
Ajax files: 9
Templates: 3
Migrations: 17
```

---

## 2. Есть ли очевидные архитектурные проблемы?

Например:

* слишком большие template.php;
* SQL-запросы внутри шаблонов;
* отсутствие обязательных файлов компонента;
* нестандартная структура модулей.

Пример:

```
⚠ component template larger than 500 lines

⚠ SQL query inside template.php

⚠ component without .description.php
```

---

## 3. Есть ли потенциально опасные конструкции?

Например:

```
⚠ $_REQUEST usage

⚠ eval()

⚠ unserialize()

⚠ include(variable)
```

# Основные команды MVP

## Полный анализ проекта

```bash
bxscan full
```

или

```bash
bxscan full ./local
```

Запускает все доступные анализаторы и выводит сводный отчёт.

---

## Анализ компонентов

```bash
bxscan components
```

Проверяет:

* количество компонентов;
* наличие `class.php`;
* наличие `component.php`;
* наличие `.description.php`;
* наличие `.parameters.php`;
* размер `template.php`;
* использование SQL внутри шаблонов;
* использование `CIBlockElement::GetList()` в представлении.

---

## Анализ AJAX

```bash
bxscan ajax
```

Проверяет:

* наличие `check_bitrix_sessid();`
* подключение `prolog_before.php`;
* использование `$_REQUEST`;
* корректность JSON-ответов.

---

## Анализ модулей

```bash
bxscan modules
```

Проверяет:

* наличие `install/index.php`;
* наличие `include.php`;
* наличие каталога `lib`;
* наличие `options.php`;
* общую структуру локальных модулей.

---

## Анализ безопасности

```bash
bxscan security
```

Выполняет поиск потенциально опасных конструкций:

```
eval(
unserialize(
$_REQUEST
$_GET
$_POST
include(variable)
require(variable)
```

# Правила работы CLI

Если путь не указан:

```bash
bxscan full
```

то используется директория:

```
./local
```

Если путь указан:

```bash
bxscan full /var/www/project/local
```

то анализируется переданная директория.

---

# Что не входит в MVP

Для первой версии намеренно исключены:

* анализ инфоблоков;
* анализ агентов;
* анализ JavaScript и BX API;
* анализ Sprint Migration;
* анализ ORM;
* поиск неиспользуемого кода;
* интеграция с CI/CD;
* автоматическое исправление ошибок.

# Основная идея проекта

`bxscan` — это специализированный инструмент аудита Bitrix-проектов, который помогает разработчику быстро ответить на три вопроса:

1. Из чего состоит проект?
2. Какие архитектурные проблемы уже существуют?
3. Какие потенциально опасные места требуют внимания?

При этом главная цель создания проекта — практическое освоение языка Go через разработку полезного инженерного инструмента.
